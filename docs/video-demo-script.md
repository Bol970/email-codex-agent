# Video Demo Script

Сценарий записи учебного проекта Email Codex Agent.

Цель: примерно за 1.5-2 минуты показать локальный email workspace с AgentMail и встроенным Codex: чтение письма, все быстрые действия агента, создание черновика и контроль отправки пользователем.

## Перед записью

1. Закрыть лишние вкладки, `.env.local`, терминалы с секретами и уведомления.
2. Открыть запись экрана на отдельном мониторе.
3. Запустить автономный демо-режим:

```bash
npm run demo:record
```

Команда сама:

- соберёт production build;
- поднимет безопасный mock+demo сервер;
- откроет отдельное окно Chromium;
- постарается открыть окно на мониторе `2`;
- покажет экранные титры;
- проиграет мягкие звуковые эффекты для титров, курсора, кликов и финала;
- будет плавно двигать видимый demo-курсор к целевым кнопкам;
- откроет письмо;
- быстро пройдёт все Codex presets: `Summarize`, `Translate to Russian`, `Actions`, `Classify`, `Related`, `Follow up`, `Briefing`, `Draft reply`;
- покажет созданный черновик;
- покажет финальный титр и автоматически закроет browser.

## Роли

Пользователь:

- включает запись;
- ждёт, пока demo-runner сам пройдёт сценарий;
- после закрытия browser останавливает запись.

Codex:

- сам открывает браузер и кликает UI через Playwright;
- ведёт видимый курсор по экрану перед каждым кликом;
- сопровождает ключевые шаги короткими звуковыми эффектами;
- показывает экранные титры вместо устной озвучки;
- не отправляет письма.

## Автопилот во время записи

`npm run demo:record` делает всё сам:

1. Открывает demo browser.
2. Показывает титр с назначением проекта.
3. Плавно ведёт demo-курсор к письму `Pilot inbox: launch checklist`.
4. Открывает письмо.
5. По очереди запускает `Summarize`, `Translate to Russian`, `Actions`, `Classify`, `Related`, `Follow up`, `Briefing`.
6. Показывает разные ответы Codex для каждого действия.
7. Ведёт курсор к `Draft reply` и создаёт черновик.
8. Показывает created draft и кнопку `Send` как ручной guardrail.
9. Показывает финальный титр.
10. Закрывает browser и останавливает demo-сервер.

После автоматического закрытия browser можно останавливать запись. Если нужно оставить окно открытым для ручной проверки, запускать так: `DEMO_KEEP_OPEN=1 npm run demo:record`.
Если звук не нужен, запускать так: `DEMO_SOUND=0 npm run demo:record`.
Если звук слишком тихий, запускать так: `DEMO_SOUND_VOLUME=10 npm run demo:record`.
Если нужно ещё быстрее, запускать так: `DEMO_PACE=0.25 npm run demo:record`. Для более медленной версии: `DEMO_PACE=1 npm run demo:record`.

- Не нажимать `Send`, если это не отдельный тестовый inbox.
- Не уходить в GitHub, README или код во время записи.

Запустить `npm run demo:record` один раз без записи и проверить, что браузер сам дошёл до финального титра и закрылся.
