# Short Video Demo Script

Сценарий короткой записи учебного проекта Email Codex Agent.

Цель: за 3 минуты показать локальный email workspace с AgentMail и встроенным Codex, который помогает читать письма, суммировать их и готовить черновики без автоматической отправки.

## Перед записью

1. Закрыть лишние вкладки, `.env.local`, терминалы с секретами и уведомления.
2. Открыть запись экрана на отдельном мониторе.
3. Запустить автономный демо-режим:

```bash
npm run demo:record
```

Команда сама:

- соберёт production build;
- поднимет безопасный mock+demo сервер;
- откроет отдельное окно Chromium;
- покажет экранные титры;
- откроет письмо;
- нажмёт `Summarize`;
- нажмёт `Draft reply`;
- покажет созданный черновик.

## Роли

Пользователь:

- включает запись;
- ждёт, пока demo-runner сам пройдёт сценарий;
- после финального титра останавливает запись.

Codex:

- сам открывает браузер и кликает UI через Playwright;
- показывает экранные титры вместо устной озвучки;
- не отправляет письма.

### 0:00 - 0:20. Что это за проект

Что показать: общий вид приложения.

Текст:

> Это локальный учебный сервис Email Codex Agent. Он работает как веб-приложение на localhost, подключается к AgentMail inbox и добавляет справа встроенную панель Codex для работы с письмами.

Показать:

- список писем;
- область чтения;
- правую панель Codex.

### 0:20 - 0:50. Чтение письма

Что сделать:

1. Открыть одно входящее письмо.
2. На секунду показать тему, текст письма и область ответа.

Текст:

> Базовый workflow такой же, как в обычном почтовом клиенте: я открываю входящее письмо, читаю его содержимое, вижу thread и могу вручную ответить или сохранить черновик.

### 0:50 - 1:35. Codex summary или перевод

Что сделать:

1. Нажать `Summarize` или `Translate to Russian` в правой панели.
2. Дождаться ответа в Codex-чате.

Текст для Summarize:

> Codex получает выбранный email thread как контекст и помогает быстро понять смысл письма: кто написал, что нужно сделать, есть ли срочность и какой следующий шаг лучше выбрать.

Текст для Translate to Russian:

> Также Codex может перевести выбранное письмо на русский, сохранив смысл, имена, даты и структуру сообщения. Это полезно для быстрого разбора длинных входящих писем.

Если ответ идёт медленно:

> Ответ стримится через локальный Codex app-server, поэтому результат появляется прямо в диалоге.

### 1:35 - 2:25. Черновик ответа

Что сделать:

1. Нажать `Draft reply`.
2. Дождаться появления сообщения о создании черновика.
3. Показать карточку draft под письмом.

Текст:

Показать:

- сообщение `Черновик создан в выбранном письме`;
- текст черновика;
- кнопки `Send` и удаления draft.

### 2:25 - 2:50. Ручной контроль

Что сделать:

1. Показать поле `Write a reply`.
2. Не обязательно вводить текст; достаточно показать, что ручной ответ тоже доступен.

Текст:

> При этом приложение не заменяет пользователя. Можно самому написать ответ, сохранить draft и решить, отправлять его или нет. Codex здесь работает как помощник внутри привычного mail workflow.

### 2:50 - 3:00. Финал

Что показать: общий вид приложения.

Текст:

> В итоге это локальный email workspace: AgentMail отвечает за почту, Codex помогает с анализом и черновиками, а все необратимые действия остаются под контролем пользователя.

## Автопилот во время записи

`npm run demo:record` делает всё сам:

1. Открывает demo browser.
2. Показывает титр с назначением проекта.
3. Открывает письмо `Pilot inbox: launch checklist`.
4. Запускает `Summarize`.
5. Показывает ответ Codex.
6. Запускает `Draft reply`.
7. Показывает созданный draft.
8. Завершает финальным титром и оставляет окно открытым.

Чтобы закрыть demo browser и сервер после записи, нажать `Ctrl+C` в терминале, где запущен `npm run demo:record`.

## Что не показывать и не проговаривать

- Не рассказывать про настройки приватности.

- Не показывать ключи, терминалы с секретами или историю команд.

- Не уходить в GitHub, README или код во время короткой записи.

## Проверка перед записью

Запустить `npm run demo:record` один раз без записи и проверить, что браузер сам дошёл до финального титра.
